{\bfseries{aicpp}} is a deterministic symbolic program synthesis engine written in C++, guided by structural reductions generated by large language models (LLMs).

Instead of using LLMs to generate final solutions, aicpp uses them to {\bfseries{reduce the structural search space}}, while a native compiled symbolic engine performs bounded-\/depth exhaustive search.

The goal is to explore a hybrid architecture that balances\+:


\begin{DoxyItemize}
\item Determinism ~\newline

\item Explicability ~\newline

\item Structural compositionality ~\newline

\item Native performance ~\newline

\item Controlled LLM guidance ~\newline

\end{DoxyItemize}

\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{🧠 Core Idea}{🧠 Core Idea}}\label{md_README_autotoc_md2}
LLMs are powerful pattern recognizers.

However, instead of letting them directly generate solutions, we use them to\+:


\begin{DoxyEnumerate}
\item Select relevant primitives ~\newline

\item Generate partial structural parameterizations ~\newline

\item Reduce combinatorial explosion ~\newline

\end{DoxyEnumerate}

Then a deterministic C++ engine\+:


\begin{DoxyItemize}
\item Composes typed primitives
\item Explores bounded search depth
\item Orders by cost
\item Returns explicit symbolic solutions
\end{DoxyItemize}

LLM = structural prior ~\newline
 C++ engine = deterministic solver ~\newline

\hypertarget{md_README_autotoc_md9}{}\doxysection{\texorpdfstring{🏗 Architecture Overview}{🏗 Architecture Overview}}\label{md_README_autotoc_md9}
The system consists of\+:
\begin{DoxyEnumerate}
\item Primitives (C++)
\begin{DoxyItemize}
\item Typed transformation functions
\end{DoxyItemize}
\item Neuron
\begin{DoxyItemize}
\item Wraps a primitive function
\item Defines input/output types
\end{DoxyItemize}
\item Connection
\begin{DoxyItemize}
\item Composed graph of neurons
\item Brain
\item Manages search space
\item Performs cost-\/ordered exploration
\item Serializes discovered structures
\end{DoxyItemize}
\item LLM Pipeline (Python)
\begin{DoxyItemize}
\item Analyzes ARC task examples
\item Selects relevant primitives
\item Generates structural partials
\item Triggers dynamic compilation
\item Launches exploration
\end{DoxyItemize}
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md10}{}\doxysection{\texorpdfstring{🧪 Example (\+ARC Flip Task)}{🧪 Example (ARC Flip Task)}}\label{md_README_autotoc_md10}
Given ARC input-\/output examples, the LLM selects only\+:
\begin{DoxyItemize}
\item {\ttfamily flipud}
\item {\ttfamily fliplr}
\end{DoxyItemize}

The engine then deterministically explores combinations and returns a symbolic solution such as\+: {\ttfamily flipud(fliplr(input))}

No stochastic reasoning occurs in the solving phase.

\hypertarget{md_README_autotoc_md12}{}\doxysection{\texorpdfstring{📊 Why This Approach?}{📊 Why This Approach?}}\label{md_README_autotoc_md12}
Traditional approaches\+:
\begin{DoxyItemize}
\item Deep learning → latent, non-\/explicit
\item Program synthesis → combinatorial explosion
\item LLM direct generation → non-\/deterministic
\end{DoxyItemize}

aicpp explores\+: LLM-\/guided structural reduction + deterministic symbolic completion

This separation preserves\+:
\begin{DoxyItemize}
\item Reproducibility
\item Inspectability
\item Controlled search

\hypertarget{md_README_autotoc_md25}{}\doxysection{\texorpdfstring{🧠 Vision}{🧠 Vision}}\label{md_README_autotoc_md25}
aicpp investigates a fundamental hypothesis\+: Large language models are most effective when used as structural reducers, not as direct reasoning engines.

The long-\/term goal is to build scalable, deterministic, and explicable hybrid reasoning systems.
