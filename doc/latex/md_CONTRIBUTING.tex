 Thank you for your interest in contributing to {\bfseries{aicpp}}.

aicpp is a deterministic native C++ symbolic synthesis engine guided by LLM-\/generated structural reductions. ~\newline
 The project focuses on\+:


\begin{DoxyItemize}
\item Symbolic program synthesis
\item Structural search space reduction
\item Deterministic exhaustive exploration
\item Native C++ compilation
\item Reproducibility and explicability
\end{DoxyItemize}

Before contributing, please read this document carefully.

\DoxyHorRuler{0}
\hypertarget{md_CONTRIBUTING_autotoc_md851}{}\doxysection{\texorpdfstring{Project Philosophy}{Project Philosophy}}\label{md_CONTRIBUTING_autotoc_md851}
aicpp is {\bfseries{not}}\+:


\begin{DoxyItemize}
\item A deep learning framework
\item A stochastic generative solver
\item A black-\/box ARC solver
\end{DoxyItemize}

aicpp is\+:


\begin{DoxyItemize}
\item A deterministic symbolic engine
\item Guided externally by LLM-\/based structural parameterization
\item Fully reproducible
\item Fully inspectable
\end{DoxyItemize}

Maintaining this architectural clarity is essential.

\hypertarget{md_CONTRIBUTING_autotoc_md864}{}\doxysection{\texorpdfstring{Contributions That Are Out of Scope}{Contributions That Are Out of Scope}}\label{md_CONTRIBUTING_autotoc_md864}
The following are generally not aligned with the project direction\+:


\begin{DoxyItemize}
\item Replacing the symbolic engine with neural inference
\item Introducing non-\/deterministic search mechanisms
\item Adding heavy ML frameworks
\item Refactoring without architectural justification
\item Introducing unnecessary dependencies
